Zoom 会议个人免费版使用简介

Apr 08, 2017 in tools

Zoom 是一款很不错的在线视频会议系统。总体而言，它虽然是后起之秀，但是大有赶超前辈的意思。如果你知道Zoom的核心开发团队出自WebEx，估计你就不再会惊讶它的优秀了。

本文只简单介绍一下Zoom的免费视频会议(Basic Personal Meeting)桌面客户端使用。我主要介绍Zoom.us官方网站，官网提供会议免费使用的版本，国内的代理商(zoom.cn)没有免费版本，只能免费试用7~14天。实际测试发现，其官网的视频会议速度也还不错，比较稳定。

支持手机App.

浏览器

建议使用Chrome、Firefox、Safari浏览器。实际使用发现，Edge浏览器有问题，也不建议使用IE(未测试)。使用Edge在网页上开启会议，会出现界面空白，无法下载客户端软件，

官网介绍（zoom.us)

打开官网，如果要发起会议，需要注册一个账号，加入其他人的会议则不需要注册。
登录账号后，在网页顶部右侧有三个导航菜单链接：Schedule a Meeting, Join a Meeting, Host a Meeting.
你可以计划安排一个会议，比如下周的某一天几点开始会议，或者加入别人的会议，也可以主持自己的会议(Host a Meeting).


如果你第一次使用，没有安装Zoom桌面客户端，页面会提示并自动下载Zoom 客户端。
当你安装了Zoom客户端之后，这几个功能，都可以在客户端上完成。

下载安装桌面客户端

视频会议最好还是使用客户端进行，最好不要用浏览器（浏览器需要安装插件，未测试）
你也可以直接在官网上自己手动下载客户端并安装。安装后使用客户端而非浏览器操作。
网址：zoom.us/meetings

桌面客户端

登录客户单App主界面如下，可以计划、加入和建立（主持Host)会议，共享（屏幕、白板）。在Settings中可设置常规选项、音频、视频、录制等。

进入会议后界面主要有菜单、工具栏、视频或共享主窗口三部分。
菜单栏中可进行设置、会议操作，工具栏功能与菜单栏基本相同。

发起会议

客户端主界面(Home)

  • Start with/without Video - 立即开始会议。
  • Schedule - 计划会议
  • Share screen - 共享屏幕

加入会议

两种方式都可以

  1. 客户端主界面(Home)，点击Join，输入Meeting Id.
  2. 点击发起会议的邀请链接，例如https://zoom.us/j/742625326，最后的数字即Meeting Id. 点击链接后，一般系统会用浏览器打开，然后页面提示下载安装或启动已有的客户端程序，点击允许(Allow)继续。这样就无需注册自动加入会议了。注意 ，前面提到过不要用edge或IE。点击链接，系统或找默认的浏览器，如果是edge或IE浏览器打开了，而无法下载或启动客户端程序，请复制这个邀请链接到Chrome、Firefox、Safari浏览器中。已安装客户端的，可以采用方法1，输入链接最后的数字（Meeting id)。

邀请其他人加入会议

在会议界面底部的工具栏，点击Invite，或者菜单栏Meeting->Invite，复制邀请链接，或发送到email.
也可以直接把Meeting Id发给其他人，Meeting Id在会议界面顶部显示。

会议主要功能

音频

  • 设置输入输出设备
  • 静音

视频

  • 视频设置
  • 切换视图
  • 锁定视频窗口(Pin Video)

共享屏幕

  • 屏幕
  • 白板
  • 平板或手机
  • 程序（如打开的幻灯片软件）

设置主持人(Make Host)

当前主持人，能设置其他人为主持人。会议主持人的控制功能最多，比如主持人可以录制视频。
在每个人的视频窗口，点击鼠标，出现More，在下拉菜单中点击Make Host。或者在底部工具栏，单击Manage Participants，在鼠标放在参加者名字上，出现More，在下拉菜单中点击Make Host.

聊天

点击底部工具栏Chat，打开聊天窗口，可向所有人(Everyone)或者指定人发消息。
免费版本不支持发文件。

录制视频(Record)

主持人可以录制视频。点击底部工具栏的Record，开始录制本地视频（服务器录制视频免费版不提供）。停止录制视频后，提示结束会议后自动保存。结束会议后，选择视频保存位置，完成录制。

常见一些问题

  • 主要问题仍然是IE和Edge浏览器的问题，换成其他浏览器。
  • 没有声音，连接音频时选择电脑，不要选择手机呼入。

Mantis Email 设置

Mar 25, 2017 in Tech

本文介绍mantis邮件配置，以实现用户邮件注册、邮件通知等功能。第一次部署mantis 邮件就没有配置成功，无法注册用户。当时改成管理员添加用户，但是非常不方便。只加了一个test用户进行提交问题，这样就不知道具体是谁提交的了。最近又在另外一台服务器上部署mantis，而且这次软件使用人员比第一次多，所以还是用户自行注册比较好，也知道哪个用户提交的问题，同时，管理员、开发人员和用户都可以接收关于bug状态的邮件通知，这才能更好的发挥一个Issue Tracking System的功能。

我选择的是最新版，2.0.1。mantis安装这里不做介绍，只介绍邮件配置，其他像匿名登录配置和一些自定义配置由于比较简单，这里不提及。

配置

以163邮箱为例，并经过验证。其他邮箱未测试。

config_inc.php

这里需要注意的一点是，你的163邮箱账号设置里，需要开启smtp功能，这里的

; For Win32 only.
; http://php.net/smtp
SMTP = smtp.163.com
; http://php.net/smtp-port
smtp_port = 25
```

你能看到smtp_port = 25在config_inc.php和php.ini中都设置了，我没有测试只在一个地方设置是否可以。

httpd.conf

最后在apache的httpd.conf文件中，找到listen 这里，换成服务器的ip:port，这样发给用户注册邮件的链接，就不是localhost了。

你可以自己通过php脚本测试邮件配置，或者通过打印日志的方式。
