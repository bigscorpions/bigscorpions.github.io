Mantis Email 设置

Mar 25, 2017 in Tech

本文介绍mantis邮件配置，以实现用户邮件注册、邮件通知等功能。第一次部署mantis 邮件就没有配置成功，无法注册用户。当时改成管理员添加用户，但是非常不方便。只加了一个test用户进行提交问题，这样就不知道具体是谁提交的了。最近又在另外一台服务器上部署mantis，而且这次软件使用人员比第一次多，所以还是用户自行注册比较好，也知道哪个用户提交的问题，同时，管理员、开发人员和用户都可以接收关于bug状态的邮件通知，这才能更好的发挥一个Issue Tracking System的功能。

我选择的是最新版，2.0.1。mantis安装这里不做介绍，只介绍邮件配置，其他像匿名登录配置和一些自定义配置由于比较简单，这里不提及。

配置

以163邮箱为例，并经过验证。其他邮箱未测试。

config_inc.php

这里需要注意的一点是，你的163邮箱账号设置里，需要开启smtp功能，这里的

; For Win32 only.
; http://php.net/smtp
SMTP = smtp.163.com
; http://php.net/smtp-port
smtp_port = 25
```

你能看到smtp_port = 25在config_inc.php和php.ini中都设置了，我没有测试只在一个地方设置是否可以。

httpd.conf

最后在apache的httpd.conf文件中，找到listen 这里，换成服务器的ip:port，这样发给用户注册邮件的链接，就不是localhost了。

你可以自己通过php脚本测试邮件配置，或者通过打印日志的方式。

osx分区合并已删除的分区的方法

Jun 18, 2016 in Tech

Macbook pro安装了除了osx系统外，又安装了一个archlinux 系统。各占249G的分区。后osx的分区空间不足了，开始需要压缩archlinux的分区给osx. 但是跟osx相邻的分区为arch的主分区，不能压缩了。这样除了删掉arch的分区，没有办法。在osx里用disk utility删除arch分区，但是发现仍然无法于osx分区合并。可能是不应该用disk utility删除arch分区。最后解决方法不是很快速，但好在保证了原osx系统的完整性。
解决方法如下:

  • 利用Time Machine备份整个OS X系统.
  • 重启电脑，按command+r，进入OS X recover 系统，进入后利用Disk Utility 删除原osx分区。
  • Restore from Time Machine Backup. (第一次恢复，中间出错，重启后重新恢复，成功完成了恢复）.

恢复后，osx系统基本还算完整，初步看，文件和应用都没有问题。当然，archlinux无法保住了，因为当时的分区不是很合理，应该把非arch的主分区跟osx分区相邻，这样就可以通过压缩该分区释放空间给osx.
